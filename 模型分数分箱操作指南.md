# 模型分箱与策略阈值方法论

## 一、核心思路

> **等频初分 → 相邻箱合并 → 跨期验证 → 逐分数绘制阈值曲线 → 在风险、规模和收益约束下确定通过、审核、拒绝阈值。**

需要区分两个概念：

* **分箱**：建立稳定、单调、可解释的风险等级；
* **阈值**：决定哪些客户通过、人工审核或拒绝。

阈值可以落在分箱边界上，也可以先在细粒度分数上搜索，再对齐到可上线的分数边界。

---

## 二、分箱方法

### 1. 样本划分

按时间将样本划分为：

* **样本内**：用于设计分箱和选择候选阈值；
* **OOT 样本**：用于最终验证，不能重新分箱或反复调参。

只使用标签已经成熟的样本计算逾期率。

### 2. 等频初分

在样本内上按模型分进行**等频 20 箱**：

* 每箱样本量尽量接近；
* 同分较多时允许实际箱数少于 20；
* 首尾边界设置为 `-∞` 和 `+∞`，避免线上样本落在区间之外。

### 3. 计算分箱指标

每箱至少计算：

* 样本量、坏样本量；
* 箱内逾期率；
* Lift；
* 逾期率标准误或置信区间；
* 累计通过率、累计逾期率；
* 样本内和 OOT 的箱占比及逾期率。

### 4. 合并相邻箱

按以下顺序合并：

1. 样本量或坏样本量过少的箱；
2. 逾期率明显倒挂且差异不显著的箱；
3. 相邻箱逾期分布差异不显著的箱；
4. 跨期表现不稳定的箱。

可使用 **ChiMerge、两比例检验和逾期率置信区间**辅助判断。

最终一般保留 **6～10 个风险等级**，并满足：

* 逾期率整体单调；
* 单箱样本充分；
* 相邻等级存在风险差异；
* OOT 风险排序基本稳定；
* 边界便于解释和系统配置。

---

## 三、阈值曲线方法

假设模型为**高分高风险**，候选阈值为 (c)，则通过人群为：

[
score \leq c
]

候选阈值可以使用：

* 每个模型分唯一值；
* 固定分数步长；
* 分数分位点；
* 最终分箱边界。

对每个候选阈值，从低风险端向高风险端累计计算：

| 指标        | 作用       |
| --------- | -------- |
| 累计通过率     | 衡量业务规模   |
| 累计逾期率     | 衡量整体风险   |
| 边际逾期率     | 衡量新增客群风险 |
| 累计金额损失率   | 衡量本金损失   |
| EL        | 衡量预期损失   |
| 风险后收入或 UE | 衡量经济价值   |
| 审核量       | 衡量人工产能压力 |

其中，**边际风险**非常重要：当阈值继续放宽时，如果新增客群逾期率快速上升、边际收益转负，通常意味着已经接近合理阈值。

---

## 四、策略阈值确定

采用三段式策略：

```text
自动通过：score <= 自动通过阈值
人工审核：自动通过阈值 < score <= 拒绝阈值
拒绝：score > 拒绝阈值
```

阈值不能仅按箱数比例机械切分，而应在以下约束下优化：

* 逾期率或 EL 不超过风险上限；
* 通过率或放款规模达到业务目标；
* 风险后收入或总 UE 不低于经营底线；
* 新增客群的边际 UE 为正；
* 人工审核量不超过团队产能；
* OOT 和压力情景下仍可接受。

最终建议输出三套方案：

| 方案   | 主要目标         |
| ---- | ------------ |
| 增长方案 | 在风险底线内最大化通过率 |
| 平衡方案 | 平衡通过率、风险和收益  |
| 保守方案 | 优先控制逾期率和损失   |

正式推荐方案应选择满足约束且位于**风险—规模—收益 Pareto 前沿**的阈值组合，而不是只追求最低逾期率或最高通过率。

---

## 五、验证与上线

最终分箱和阈值必须在 OOT 上复用完全相同的边界，重点验证：

* 各风险等级逾期率排序是否稳定；
* 通过率、逾期率和 EL 是否符合预期；
* PSI、AUC、KS 是否出现明显恶化；
* 不同月份、产品、渠道是否存在局部失效；
* 人工审核量是否可执行。

最终上线配置应明确：

* 模型版本；
* 分数方向；
* 分箱真实切点；
* 区间左右开闭规则；
* 自动通过和拒绝阈值；
* 标签与样本时间范围；
* 生效时间和回滚版本。

## 六、一句话总结

> **分箱负责把模型分整理成稳定的风险等级；阈值负责沿分数逐步放宽准入，在风险、规模、收益和审核产能之间寻找最优决策点。**
