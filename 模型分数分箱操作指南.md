# 模型分数分箱操作指南

## 一、最终执行方案

现行流程建议统一为：


> **统一数据口径 → 按时间划分策略调优集与 OOT → 等频 20 箱初分 → 检查样本量、坏样本量、单调性及相邻箱差异 → ChiMerge 合并为 6～10 个风险等级 → 在细粒度阈值上计算通过率、坏账率和边际风险 → 形成增长、平衡、保守三套策略 → 使用 OOT 确认 → 输出可上线的分箱和阈值配置。**

### 三条基本原则

1. **分箱用于观察风险和收益曲线，阈值用于做业务决策，两者不能混为一谈。**阈值可以落在分箱边界上，也可以基于更细粒度分数逐点搜索。
2. 对模型分分层时，应优先观察**实际坏账率、校准后 PD、累计 EL 和收益曲线**；WOE、IV 更适合特征分箱，在模型分层中仅作为辅助参考。
3. 线上决策应使用预测时点可获得的预计收益；成熟样本回测应使用实际现金流，并反向校准收益模型。预测和回测服务于不同环节，并不冲突。

### 2.2 建议统一到数据库的数据血缘


推荐生成一张标准分箱样本宽表：

| 标准字段 | 来源 | 说明 |
|---|---|---|
| `application_id` | `res/aus_old_risk_bid_mltmodel_v1_2_20260325_lgb_score.csv`、`res/application_info.csv` | 唯一关联键 |
| `sample_datetime` | 优先使用评分表 `sample_datetime`，或申请信息表 `application_time` | 用于策略调优集和 OOT 时间切分 |
| `score` | `aus_old_risk_bid_mltmodel_v1_2_v20260325_lgb_score` 列 | 原始模型分，不应使用贷后信息计算 |
| `duedate_3m_30` | `res/application_info.csv` | 主坏账标签，空值代表未成熟或未知 |
| `duedate_1m_30` | `res/application_info.csv` | 辅助早期标签，用于稳定合并 |
| `principal` | `res/application_info.csv` | 原始放款本金 |
| `estimate_principal_remaining_mob3` | `res/application_info.csv` | MOB3 剩余本金 |
| `dpd_days_ever_mob3` | `res/application_info.csv` | MOB3 内最大历史逾期天数 |





建议 Python 关联代码：

```python
import pandas as pd

SCORE_PATH = "res/aus_old_risk_bid_mltmodel_v1_2_20260325_lgb_score.csv"
SCORE_COL = "aus_old_risk_bid_mltmodel_v1_2_v20260325_lgb_score"
INFO_PATH = "res/application_info.csv"
SAMPLE_START_DATE = "2024-01-01"

score_df = pd.read_csv(SCORE_PATH)
info_df = pd.read_csv(INFO_PATH, parse_dates=["application_time"])

info_df = info_df[info_df["application_time"] >= SAMPLE_START_DATE]

data = score_df.merge(
    info_df[[
        "application_id",
        "application_time",
        "duedate_3m_30",
        "duedate_1m_30",
        "principal",
        "estimate_principal_remaining_mob3",
        "dpd_days_ever_mob3",
    ]],
    on="application_id",
    how="inner",
)

n_score_only = score_df["application_id"].nunique() - data["application_id"].nunique()
n_info_only = (info_df["application_id"].nunique()
               - data["application_id"].nunique())

print(f"评分表样本数: {score_df['application_id'].nunique()}")
print(f"表现表样本数: {info_df['application_id'].nunique()}")
print(f"成功关联样本数: {data['application_id'].nunique()}")
print(f"评分表未匹配数: {n_score_only}")
print(f"表现表未匹配数: {n_info_only}")
print(f"重复 application_id 数: {data['application_id'].duplicated().sum()}")
```

`SCORE_PATH`、`INFO_PATH`、`SAMPLE_START_DATE` 需在代码配置中填写，不能长期写死在脚本中。更换模型版本时需更新评分 CSV 文件名和对应列名。

### 2.3 数据质量检查

分箱前必须输出以下检查结果：

1. `application_id` 是否唯一，重复申请如何保留；
2. 评分表、表现表及关联后的样本数，左右表未匹配数量；
3. 分数缺失率、最小值、最大值、唯一值数量和同分占比；
4. 标签是否只包含 `0/1/NULL`；
5. 主标签成熟率，以及调优集和 OOT 的成熟率；
6. `sample_datetime` 是否真实代表申请或评分时点；
7. 本金、剩余本金和逾期字段的缺失及异常值；
8. 是否存在使用贷后信息生成模型分的数据泄漏。

主标签为 NULL 的样本不得进入坏账率计算。辅助标签缺失也不得直接填为 0，否则会把未知样本误判为好样本。


### 2.4 完整样本体系、成熟度与审批偏差

仅有“调优集 + OOT”可以运行现行代码，但标准方法论建议区分以下样本：

| 样本 | 主要用途 | 使用要求 |
|---|---|---|
| 训练集 | 训练模型 | 不直接用于选择策略阈值 |
| 校准集 | 将模型分映射为目标期限 PD | 样本充足时与策略调优集分开 |
| 策略调优集 | 设计分箱、合并等级、选择候选阈值 | 与训练集独立 |
| OOT 样本 | 在更晚时间段做最终确认 | 原则上只验证，不反复调参 |
| 线上成熟样本 | 回测真实坏账、现金流和 UE | 用于持续校准和策略迭代 |

样本量不足时，可以合并校准集和策略调优集，但应使用交叉验证、Bootstrap 或滚动时间验证评估不确定性。

#### 标签成熟度

优先使用完整表现窗口。若数据尚未成熟，只能使用 FPD7、FPD30 等早期指标时，需要：

1. 验证早期指标与最终损失之间的映射是否稳定；
2. 按产品、期限、渠道和月份重复验证；
3. 保存映射模型、参数版本、验证结果和更新时间；
4. 在样本成熟后用真实结果回溯修正。

#### 历史审批偏差

若历史样本仅包含已通过并已放款客户，直接用其评估新的拒绝阈值会受到审批选择偏差影响。优先使用：

- 全量申请人的外部风险表现；
- 随机审批或小流量探索样本；
- 历史政策变化形成的准实验样本；
- 必要时使用拒绝推断，但只作为敏感性分析，不能作为唯一依据。

### 2.5 风险标签与经济口径的前置统一

在分箱和阈值测算前，必须先统一以下定义，否则不同团队可能得到完全不同的结论。

#### 风险标签

至少明确：

- 违约定义：例如 FPD30、30+ DPD、90+ DPD、核销或其他公司统一口径；
- 表现窗口：放款后 30 天、90 天、6 个月或全生命周期；
- 统计单位：申请、客户、账户还是贷款合同；
- 是否包含提前结清、展期、重组、欺诈和重复申请；
- 未成熟样本如何排除或进行成熟度调整。

不同模型和产品只有在**违约定义与预测期限一致**时，PD 才具有直接可比性。

#### 经济口径

需要统一：

- PD、LGD、EAD 的定义和预测期限；
- 资金成本、获客成本、数据成本、运营成本、催收成本和资本成本；
- 利息、手续费、罚息、税费和回收金额的处理方式；
- 是评估合同期收益、年度收益，还是全生命周期收益；
- UE 是按申请、按获批账户、按放款账户，还是按单位本金计算。

## 四、直接执行流程

### 步骤 1：构建标准样本

按 `application_id` 关联模型分与贷后表现，保留模型分、申请时点、主标签、辅助标签和金额字段。

关联后至少输出：

```text
评分表样本数
表现表样本数
成功关联样本数
评分表未匹配数
表现表未匹配数
重复 application_id 数
```

不要只使用 `inner join` 后的结果而不检查被丢弃的样本。

### 步骤 2：划分策略调优集与 OOT

现行代码按时间切分：

```python
tuning = data[data["sample_datetime"] < oot_cut]
oot = data[data["sample_datetime"] >= oot_cut]
```

要求：

- 调优集用于分箱、合并和候选阈值选择；
- OOT 只用于最终确认，不应反复查看后继续调参；
- 只保留主标签已成熟的样本计算风险表现；
- OOT 必须位于调优集之后；
- 若根据 OOT 修改方案，该 OOT 已变成调优样本，需要再保留更晚的独立样本验证。

### 步骤 3：等频 20 箱初分

在策略调优集上执行：

```python
bin_series, bin_edges = pd.qcut(
    tuning[score_col],
    q=20,
    duplicates="drop",
    retbins=True,
)
```

`duplicates="drop"` 用于处理大量同分造成的重复切点。实际箱数少于 20 时，应记录同分情况，不应强行制造切点。

首尾切点必须改为开口边界：

```python
bin_edges[0] = -np.inf
bin_edges[-1] = np.inf
```

这样 OOT 和线上样本即使超出调优集的分数范围，也不会被遗漏。

### 步骤 4：计算初始分箱指标

每箱至少输出：

| 指标 | 计算方式 | 用途 |
|---|---|---|
| 样本量 `n` | 该箱主标签有效样本数 | 判断统计充分性 |
| 坏样本数 `B` | `SUM(primary_label)` | 判断坏样本是否稀疏 |
| 坏账率 | `B / n` | 核心风险排序指标 |
| 标准误 `SE` | `sqrt(r(1-r)/n)` | 判断坏账率波动 |
| 累计通过率 | 低风险端累计样本 / 总样本 | 阈值分析 |
| 累计坏账率 | 低风险端累计坏样本 / 累计样本 | 阈值分析 |
| Lift | 箱坏账率 / 总体坏账率 | 风险浓度 |
| WOE、IV | 好坏分布差异 | 辅助参考 |
| AUC、KS | 全量分数排序能力 | 模型区分能力 |
| Spearman | 箱序与坏账率的秩相关 | 分箱单调性 |

WOE 出现零好样本或零坏样本时应使用统一平滑项，例如 0.5，不应直接将无穷值填成 0。


#### 关键统计公式

设初始分为 20 箱，第 `k` 个等频切点为：

\[
c_k=Q_{score}\left(\frac{k}{20}\right),\qquad k=1,2,\ldots,19
\]

设第 `i` 箱总样本数为 \(n_i\)，坏样本数为 \(B_i\)，则：

\[
r_i=\frac{B_i}{n_i}
\]

\[
SE(r_i)=\sqrt{\frac{r_i(1-r_i)}{n_i}}
\]

比较相邻两箱风险率时，可使用两比例差异近似检验：

\[
SE(r_1-r_2)=\sqrt{\frac{r_1(1-r_1)}{n_1}+\frac{r_2(1-r_2)}{n_2}}
\]

\[
z=\frac{r_1-r_2}{SE(r_1-r_2)}
\]

若差异较小、置信区间高度重叠，且不存在业务上的明显差异，可考虑合并。

#### ChiMerge 原理

ChiMerge 比较相邻两箱的好坏分布：

\[
\chi^2=\sum_{i=1}^{2}\sum_{j=1}^{2}\frac{(A_{ij}-E_{ij})^2}{E_{ij}}
\]

其中：

\[
E_{ij}=\frac{R_i\times C_j}{N}
\]

卡方值越小、`p` 值越大，说明两箱好坏分布越接近，越适合合并。期望频数过小时，应考虑精确检验、统一平滑或直接合并稀疏箱。

#### WOE 与 IV

对模型分层而言，WOE 和 IV 仅作为辅助，不应替代 AUC、KS、风险单调性、校准和经济价值。

设第 `i` 箱坏样本数为 \(B_i\)、好样本数为 \(G_i\)，全体坏样本数为 \(B\)、好样本数为 \(G\)，采用“坏 / 好”方向：

\[
WOE_i=\ln\left(\frac{B_i/B}{G_i/G}\right)
\]

\[
IV=\sum_i\left(\frac{B_i}{B}-\frac{G_i}{G}\right)\times WOE_i
\]

| IV 区间 | 常见解释 |
|---|---|
| `< 0.02` | 区分能力很弱 |
| `0.02～0.10` | 较弱 |
| `0.10～0.30` | 中等 |
| `0.30～0.50` | 较强 |
| `> 0.50` | 很强，需要检查稳定性、样本偏差和潜在泄漏 |

不同机构可能采用相反的 WOE 符号定义，代码、报告和历史结果必须保持一致。

### 步骤 5：检查并合并分箱

推荐按以下顺序处理：

1. 空箱、极小样本箱和坏样本稀疏箱；
2. 大量同分造成的边界异常；
3. 局部坏账率倒挂；
4. 相邻箱卡方或两比例 Z 检验差异不显著；
5. 调优集与 OOT 的分布和风险排序；
6. 最后才做有限的边界取整和业务适配。

现行 ChiMerge 规则：

- 初始 20 箱；
- 超过 `max_bins=10` 时，优先合并风险分布最接近的相邻箱；
- 箱数降至 10 以内后，仅在以下情形继续合并：
  - 所有指定标签的相邻箱差异均不显著；
  - 存在局部倒挂；
  - 任一箱样本量不足；
  - 任一箱主标签坏样本数不足；
- 最低保留 `min_bins=6` 个箱。

对于多个标签，建议每个标签分别计算相邻箱检验。不能将缺失辅助标签填为 0；应在该标签自己的有效样本范围内计算好坏分布。

#### 倒挂处理原则

坏账率倒挂不能一律强制合并：

- 若倒挂箱样本少、置信区间高度重叠或差异不显著，可视为抽样噪声并合并；
- 若倒挂差异显著，或只在特定产品、渠道、月份出现，应先排查数据、分群和模型失效，不应通过合并掩盖结构性问题。

#### 最终边界要求

最终策略阈值必须使用**真实切点**，不能使用每箱样本中观测到的 `score_max` 或 `score_min` 代替。否则线上出现训练样本中未观察到的分数时，可能产生区间空档。

同时统一区间语义，例如：

```text
(-∞, 0.10]
(0.10, 0.18]
(0.18, 0.25]
...
```

报告、JSON 配置和线上判断必须使用完全相同的左右开闭规则。

### 步骤 6：应用到 OOT

OOT 必须复用调优集确定的最终切点，禁止重新 `qcut`。

至少验证：

| 维度 | 指标 |
|---|---|
| 分布稳定性 | 各箱占比、PSI |
| 风险排序 | 各箱坏账率、Spearman、AUC、KS |
| 概率校准 | 平均 PD、实际坏账率、O/E、Brier Score；有 PD 时执行 |
| 策略表现 | 通过率、坏账率、金额损失、审核量 |
| 分群表现 | 月份、产品、渠道和关键客群 |

PSI 可参考：`<0.10` 相对稳定，`0.10～0.25` 需要观察，`>0.25` 需要重点排查。该区间只是经验预警，不是自动下线标准。

OOT 某箱样本为 0 时仍应保留该箱并按 0 占比计算 PSI，不能因为删除空箱而放弃 PSI。

### 步骤 7：PD 校准

在生成累计阈值曲线和经济指标前，应先将模型分映射为目标期限的 PD。校准后的 PD 是 EL、收入和 UE 计算的必要输入。

#### 7.1 为什么需要 PD 校准

- 原始模型分通常只反映相对排序，不代表真实违约概率；
- EL = PD × LGD × EAD，其中 PD 必须是概率尺度，不能直接用原始分数代入；
- 校准后的 PD 可在不同模型、产品和期限之间直接比较；
- 监管和内部报告通常要求输出可解释的概率值而非抽象分数。

#### 7.2 校准方法

| 方法 | 原理 | 优点 | 缺点 |
|---|---|---|---|
| Platt Scaling | 用逻辑回归将分数映射为概率 | 简单、稳定、样本需求小 | 假设单调 S 形关系 |
| Isotonic Regression | 非参数保序回归 | 不假设函数形式 | 需要更多样本，尾部不稳定 |
| 分箱平均 | 每箱实际坏账率直接作为该箱 PD | 直观、易解释、无模型假设 | 箱内同质假设，边界不连续 |
| Beta 校准 | 在 Platt 基础上增加形状参数 | 更灵活的概率映射 | 参数估计更复杂 |

推荐流程：

1. 在校准集（样本充足时应与策略调优集分开）上拟合校准模型；
2. 在 OOT 上验证校准效果（O/E 比、Brier Score、可靠性图）；
3. 保存校准模型参数、版本和拟合时间。

#### 7.3 无 PD 校准数据时的替代方案

若当前没有独立的 PD 校准模型或外部 PD 基准，可采用以下降级方案：

| 层级 | 方案 | 适用条件 |
|---|---|---|
| L1：完整校准 | 使用校准集拟合 Platt / Isotonic，OOT 验证 | 有独立校准集或样本量充足 |
| L2：分箱 PD | 以最终分箱的实际坏账率作为该分数段的经验 PD | 样本量一般，分箱稳定 |
| L3：排序标记 | 仅输出分数排序和坏账率，标注"PD 未校准" | 样本量不足或标签未成熟 |

当前代码可采用 **L2（分箱 PD）** 作为起步方案：在各最终分箱内计算观察坏账率，直接作为该分数段的经验 PD 用于后续 EL 和 UE 估算。报告中需明确标注 PD 来源为"分箱观察坏账率"而非"独立校准 PD"。

#### 7.4 校准验证

无论采用哪种方案，至少输出：

| 验证项 | 指标 | 说明 |
|---|---|---|
| 整体校准 | O/E 比 | `O/E ≈ 1` 表示平均预测与实际情况一致 |
| 逐箱校准 | 各箱预测 PD vs 实际坏账率 | 检查是否存在系统性高估或低估 |
| 校准 sharpness | Brier Score | 越低越好，衡量概率预测的整体质量 |
| 校准曲线 | 可靠性图（Reliability Diagram） | 可视化预测概率与实际频率的关系 |

O/E 比在 0.8～1.25 之外时，优先执行概率再校准而非直接调整分数阈值。

#### 7.5 数据要求与当前状态

| 数据项 | 用途 | 当前状态 |
|---|---|---|
| 模型分 + 真实违约标签 | 拟合校准模型或计算分箱坏账率 | 已有 |
| 独立校准集（与调优集时间分离） | 无偏校准 | 可复用调优集，标注限制 |
| 外部 PD 基准或行业违约率 | 跨模型对标 | 待接入 |
| 分产品 / 分期限 / 分渠道 PD | 分群校准 | 待接入 |
| 更长表现期标签（如 6M/12M） | 长期 PD 校准 | 待接入 |

### 步骤 8：生成累计阈值曲线与经济指标

当前模型为高分高风险，因此每个候选阈值 `c` 的通过人群为：

```text
score <= c
```

候选阈值建议包含：

- 每个最终分箱边界；
- 分数等分位点；
- 业务系统允许配置的固定步长；
- 样本量允许时的每个唯一分数。

每个阈值至少输出：

| 指标 | 定义 |
|---|---|
| 累计通过率 | 阈值内样本数 / 全量样本数 |
| 累计坏账率（笔数） | 通过人群坏样本 / 通过人群主标签有效样本 |
| 边际坏账率 | 相比上一个阈值新增客群的坏账率 |
| 累计坏账率（金额） | 坏样本 MOB3 剩余本金 / 有效样本原始本金 |
| 预测 PD | 通过人群平均或金额加权 PD |
| EL | `SUM(PD × LGD × EAD)` |
| 风险后收入 | 预计收入－EL |
| 总 UE | 逐笔 UE 加总 |
| 边际 UE | 新增分数段带来的增量 UE |
| 审核量 | 进入人工审核区的申请数 |

金额坏账率建议定义为：

```text
SUM(estimate_principal_remaining_mob3 WHERE dpd_days_ever_mob3 >= 30)
/
SUM(principal)
```

但必须要求 `dpd_days_ever_mob3`、剩余本金和本金均非空且金额大于 0。当前代码中先将逾期字段比较为布尔值，会使 NULL 被当作未逾期，需修正。


#### 经济指标计算方法

以下指标是阈值优化的核心输入。根据数据就绪程度，可采用不同层级的实施方案（见 8.5 节）。

##### 8.1 预期损失 EL

单笔预期损失：

\[
EL_j=PD_j\times LGD_j\times EAD_j
\]

| 参数 | 含义 | 常用估计方式 |
|---|---|---|
| PD | 目标期限内违约概率 | 校准后的模型 PD，或稳定分箱的实际坏账率 |
| LGD | 违约后的损失比例 | `1 - 折现后回收率`，并考虑催收费用 |
| EAD | 违约时风险暴露 | 违约时剩余本金及其他暴露 |

应明确采用 30 天、90 天、12 个月还是 Lifetime EL；不同期限的 EL 不能直接混用。

**数据要求与当前状态：**

| 数据项 | 用途 | 当前状态 |
|---|---|---|
| 校准后 PD（或分箱坏账率） | EL 计算的概率输入 | 分箱坏账率可用，独立校准 PD 待接入 |
| LGD 参数（分产品/分期限） | 违约损失比例 | 待接入，当前可暂用行业经验值或固定假设 |
| EAD 参数或剩余本金曲线 | 违约时风险暴露 | `estimate_principal_remaining_mob3` 可作近似 |
| 回收率与催收费用 | LGD 校准 | 待接入 |

**无完整数据时的降级方案：**

- LGD 缺失时：使用固定假设（如 80%），并在报告中标注假设值；
- EAD 缺失时：以 MOB3 剩余本金 `estimate_principal_remaining_mob3` 近似；
- 每笔 EL 聚合到阈值级别：`累计 EL = SUM(PD × LGD × EAD) / 总笔数` 或 `SUM(PD × LGD × EAD) / SUM(principal)`。

##### 8.2 预计收入与风险后收入

等额本息产品的合同月供：

\[
A=P\times\frac{r(1+r)^n}{(1+r)^n-1}
\]

合同总利息：

\[
Interest_{contract}=A\times n-P
\]

但合同利息不等于预计可实现收入。预计收入还应考虑：

- 违约造成的未收利息；
- 提前还款造成的利息减少；
- 实际放款率和提款率；
- 手续费退还、减免和合规限制；
- 现金流发生时间和折现。

推荐使用预计现金流口径：

\[
风险后收入=预计可收利息及费用-预计本金损失
\]

若预计收入模型已经包含违约和提前还款影响，不得再次重复扣减相同损失。

**数据要求与当前状态：**

| 数据项 | 用途 | 当前状态 |
|---|---|---|
| 合同期限、利率、本金 | 计算合同现金流 | `principal` 已有，利率和期限待接入 |
| 提前还款率（分产品/分期限） | 调整预计利息收入 | 待接入 |
| 实际放款率与提款率 | 从申请到实际生息的转化 | 待接入 |
| 手续费收入及退还规则 | 收入端完整核算 | 待接入 |
| 折现率 | 现金流折现 | 待接入，可暂用资金成本近似 |

**无完整数据时的降级方案：**

- 收入数据缺失时：以 `principal × 平均利率 × 平均期限` 估算合同利息；
- 提前还款率缺失时：暂按 0 处理，标注收入可能被高估；
- 报告中必须列出所有假设及取值。

##### 8.3 单位经济价值 UE

\[
UE=预计收入-EL-资金成本-获客成本-数据成本-运营成本-催收成本-资本成本
\]

建议同时输出：

- 总 UE（通过人群 UE 之和）；
- 单笔 UE（总 UE / 通过笔数）；
- 单位本金 UE（总 UE / 总本金）；
- 边际新增客群 UE（新增分数段带来的增量 UE）；
- UE 的置信区间或压力测试结果。

**数据要求与当前状态：**

| 成本项 | 含义 | 当前状态 |
|---|---|---|
| 资金成本 | 放款资金的加权平均成本 | 待接入 |
| 获客成本 | 单笔申请的平均获客费用 | 待接入 |
| 数据成本 | 征信、反欺诈等三方数据费用 | 待接入 |
| 运营成本 | 系统、客服、贷后管理等分摊 | 待接入 |
| 催收成本 | 按逾期阶段的分段催收费用 | 待接入 |
| 资本成本 | 监管资本或经济资本占用 | 待接入 |

**无完整数据时的降级方案：**

- 当前可先计算 **风险后收入（预计收入 − EL）** 作为简化版 UE 的近似；
- 成本数据逐项接入后逐步升级为完整 UE；
- 报告中明确区分"风险后收入"和"完整 UE"，避免混淆。

##### 8.4 审核产能约束

阈值方案不能只考虑风险和收益，还必须考虑团队的实际审核能力。

**产能模型：**

设团队每日可审核 `C` 笔申请，日申请量为 `V`，人工审核区占比为 `r_review`，则：

\[
日审核量 = V \times r_{review}
\]

\[
所需审核员数 = \frac{日审核量}{人均日审核量}
\]

**约束条件：**

\[
日审核量 \leq 可用审核产能
\]

**审核增量价值判断：**

人工审核不应简单等于中间一段分数。合理的审核对象需同时满足：

1. 模型置信度或信息完整度较低；
2. 补充资料可能显著改变风险判断；
3. 审核后的预期增量收益高于审核成本；
4. 审核量不超过团队产能。

可用以下思路判断：

\[
人工审核净价值 = 审核后预期决策收益 - 直接自动决策收益 - 审核成本
\]

只有净价值为正的客群，才适合进入人工审核区。

**数据要求与当前状态：**

| 参数 | 含义 | 当前状态 |
|---|---|---|
| 日申请量 | 每日进入模型的申请数 | 待接入 |
| 人均日审核量 | 每位审核员每日可处理量 | 待接入 |
| 团队规模 | 可用审核员数量 | 待接入 |
| 审核成本 / 笔 | 单笔人工审核的平均成本 | 待接入 |
| 审核通过率 | 人工审核后的实际通过比例 | 待接入 |

**无完整数据时的降级方案：**

- 当前仅输出各方案的预期审核量占比，作为方案对比的参考维度；
- 不执行产能硬约束，但在方案对比表中标注审核量，供业务团队评估可行性；
- 待产能数据接入后，加入硬约束：筛选出审核量 ≤ 产能的方案后再按 UE 排序。

##### 8.5 经济指标的实施分层

根据数据就绪程度，分阶段接入经济指标：

| 阶段 | 可用指标 | 所需数据 | 当前可行性 |
|---|---|---|---|
| 第一阶段 | 通过率 + 坏账率 + 边际坏账率 | 模型分 + 主标签 + 金额字段 | 当前已实现 |
| 第二阶段 | + 分箱 PD + 简化 EL（PD × 固定 LGD × 近似 EAD） | PD 校准或分箱坏账率 + LGD 假设 + 剩余本金 | 分箱坏账率可用，LGD 待设定 |
| 第三阶段 | + 完整 EL + 风险后收入 | 校准后 PD + 分产品 LGD/EAD + 利率/期限 | 待接入 |
| 第四阶段 | + 完整 UE + 审核产能优化 | 全部成本项 + 审核产能参数 | 待接入 |
| 第五阶段 | + 压力情景 + 分群 UE + 动态优化 | 历史波动率 + 分产品/渠道成本 | 待接入 |

每一阶段的输出都有独立价值：第一阶段可做风险排序和通过率决策，第二阶段可做 EL 导向的阈值选择，后续阶段逐步升级为利润最优方案。

##### 8.6 阈值优化约束

阈值不能只最大化单一指标，至少需要约束：

- 坏账率或 EL 不超过风险上限；
- 总 UE（或风险后收入）不低于经营底线；
- 通过率、放款额或收入达到最低目标；
- 人工审核量不超过团队产能；
- 关键产品、渠道和客群满足合规与公平性要求；
- 压力情景下的损失仍可承受。

### 步骤 9：确定模型风险等级

最终风险等级通常保留 6～10 个，需同时满足：

- 风险总体单调，局部异常有解释；
- 单箱样本量和坏样本量充分；
- 相邻等级具有统计或业务差异；
- OOT 风险排序基本保持；
- 边界便于系统配置、解释和回滚；
- 不同产品、渠道和月份不存在明显失效。

风险等级只是稳定的风险刻画，不等于策略决策。策略可以在等级边界上设置，也可以在更细粒度分数上搜索后再对齐到可配置边界。

### 步骤 10：确定策略阈值

现行代码采用三段式策略：

```text
自动通过：score <= auto_threshold
人工审核：auto_threshold < score <= review_threshold
拒绝：score > review_threshold
```

当前三套方案是按最终箱数比例进行启发式切分：

| 方案 | 当前代码逻辑 | 说明 |
|---|---|---|
| 保守 | 最低风险约 1/3 自动通过，前约 2/3 可进入通过范围，其余拒绝 | 风险最低 |
| 平衡 | 最低风险约 1/2 自动通过，中间审核，仅拒绝最高风险箱 | 当前默认推荐 |
| 增长 | 除最高风险箱外自动通过，最高风险箱审核，不硬拒绝 | 通过率最高 |

该逻辑适合生成初始候选方案，但最终阈值应按业务约束优化：

```text
在坏账率或 EL 不超过上限、总 UE 不低于底线、审核量不超过产能的条件下，
选择总 UE 最大或位于风险—规模—收益 Pareto 前沿的阈值组合。
```

三套正式方案至少展示：

| 方案 | 自动通过阈值 | 审核区间 | 拒绝阈值 | 通过率 | 坏账率 | EL | 总 UE | 边际 UE | 审核量 | 主要风险 |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---:|---:|
| 增长 |  |  |  |  |  |  |  |  |  |  |
| 平衡 |  |  |  |  |  |  |  |  |  |  |
| 保守 |  |  |  |  |  |  |  |  |  |  |

人工审核区不能只是机械选择中间分数段。只有补充信息可能改变决策，且审核后的增量收益大于审核成本时，才应进入人工审核。


#### 按业务目标选择阈值

| 业务目标 | 阈值选择方式 |
|---|---|
| 增长优先 | 在风险、资金和 UE 不突破底线的前提下，提高通过率 |
| 风险优先 | 在业务规模不低于底线的前提下，满足风险目标 |
| 利润优先 | 选择总 UE 或风险后总收益最大的方案，并检查边际 UE 是否为正 |
| 综合平衡 | 输出风险、规模、收益和审核量的 Pareto 前沿，由管理层选择风险偏好 |

正式方案除基准结果外，还应展示乐观和压力情景，包括 PD、LGD、资金成本、回收率和收入参数的变化。

#### 人工审核区的信息价值

人工审核不应简单等于中间一段分数。合理的审核对象通常同时满足：

1. 模型置信度或信息完整度较低；
2. 补充资料可能显著改变风险判断；
3. 审核后的预期增量收益高于审核成本；
4. 审核量不超过团队产能。

可用以下思路判断：

\[
人工审核净价值=审核后预期决策收益-直接自动决策收益-审核成本
\]

只有净价值为正的客群，才适合进入人工审核区。

## 五、效果验证与通过标准

正式上线前至少完成以下检查：

### 5.1 调优集

- 初始箱和最终箱风险曲线；
- AUC、KS、Spearman；
- 相邻箱卡方及 Z 检验；
- 样本量、坏样本量、金额暴露；
- 累计及边际坏账率；
- 有 PD 时检查校准；
- 有经济参数时检查 EL、风险后收入和 UE。

### 5.2 OOT

- 使用完全相同的分箱边界；
- 风险等级排序基本保持；
- AUC、KS 不出现无法解释的显著下降；
- PSI 和各箱占比变化可解释；
- 阈值下通过率、坏账率、EL 和审核量满足约束；
- 重要产品、渠道、月份和客群不存在明显失效。

### 5.4 PSI、校准和漂移处置

#### PSI

将分数按基准期固定分箱，设第 `i` 箱在基准期占比为 \(e_i\)，当前期占比为 \(a_i\)：

\[
PSI=\sum_i(a_i-e_i)\times\ln\frac{a_i}{e_i}
\]

| PSI 区间 | 常见判断 |
|---|---|
| `< 0.10` | 分布相对稳定 |
| `0.10～0.25` | 存在一定漂移，需要结合业务原因观察 |
| `> 0.25` | 漂移较明显，需要重点排查 |

上述范围仅是经验预警，不是监管硬标准。零占比箱应使用统一小值平滑，并记录处理方式。

#### O/E 校准

第 `i` 箱的 O/E 比：

\[
O/E_i=\frac{\sum_{j\in i}y_j}{\sum_{j\in i}PD_j}
=\frac{实际坏账率_i}{平均预测PD_i}
\]

- `O/E ≈ 1`：预测与实际较一致；
- `O/E > 1`：实际风险高于预测，模型低估风险；
- `O/E < 1`：实际风险低于预测，模型高估风险。

可将 `0.8～1.25` 作为初步预警范围，但必须结合样本量和置信区间判断。持续偏离时优先评估概率再校准，而不是直接移动分数阈值。

#### 漂移处置

| 问题表现 | 优先处置方式 |
|---|---|
| 分布漂移，但排序和校准稳定 | 观察客群结构，必要时调整经营预期 |
| 排序稳定，但整体 PD 偏高或偏低 | 概率再校准 |
| 局部客群失效 | 分群校准、增加策略规则或限制适用范围 |
| 风险偏好变化，模型仍稳定 | 调整策略阈值，不必重建模型 |
| 排序显著下降或数据关系改变 | 检查数据链路并考虑模型重建 |
| 收益或成本参数变化 | 更新经济参数后重新计算阈值 |


出现整体 PD 偏差但排序稳定时，优先重新校准；风险偏好变化但模型稳定时，仅调整阈值；排序明显下降或数据关系改变时，再考虑修复数据或重建模型。

## 附录一：主流模型分箱方法

| 方法 | 核心思想 | 优点 | 主要局限 | 适用场景 |
|---|---|---|---|---|
| 等距分箱 | 每个分数区间宽度相同 | 边界简单、易解释 | 分布偏斜时部分箱样本过少 | 分数分布较均匀、固定评分卡 |
| 等频分箱 | 每箱样本量大致相同 | 风险率估计相对稳定 | 边界不整齐，同分可能减少箱数 | 最常用的初始观察方法 |
| 固定业务分段 | 按整分、历史评级或既有政策切分 | 易管理、版本兼容 | 未必适应新模型分布 | 已有成熟策略体系 |
| 聚类分箱 | 按分数密度或聚类结果划分 | 适应非均匀分布 | 边界和解释可能不稳定 | 探索性分析 |
| 单调合并 | 合并相邻箱，使风险率总体单调 | 排序清晰、易解释 | 可能牺牲局部区分度 | 信贷风险分层 |
| ChiMerge | 按相邻箱好坏分布的卡方差异合并 | 统计依据明确、易自动化 | 对样本量和参数敏感 | 有明确好坏标签 |
| 决策树 / 熵 / MDL | 选择最能区分目标的切点 | 可识别非线性边界 | 易过拟合、边界可能碎片化 | 风险分布非线性明显 |
| 约束最优分箱 | 在单调性、箱数、样本量等约束下优化目标 | 可复现、约束完整 | 实施复杂，需防止样本内优化 | 标准化建模平台 |
| PD 校准分层 | 先校准为 PD，再按风险区间分层 | 便于跨模型和跨产品比较 | 依赖校准质量 | 多模型统一评级管理 |
| 收益 / 成本分层 | 按 EL、收益、成本或 UE 确定边界 | 与经营目标直接一致 | 依赖经济参数准确性 | 利润导向策略 |
| 混合人工方案 | 自动初分和合并后做有限业务调整 | 符合系统和经营实际 | 需严格治理，避免随意改动 | 行业常见最终方案 |

实践中通常组合使用：

> **等频初分 → 单调与统计合并 → 跨期稳定性验证 → 有限业务调整 → 逐阈值收益优化 → OOT 确认。**

## 附录二：常见误区

1. **直接把 20 个初始箱当成 20 个策略等级**：分析颗粒度不等于最终策略颗粒度。
2. **只看坏账率，不看收益和成本**：低风险不一定代表总利润最高。
3. **只看单笔 UE，不看总 UE**：可能错过具有正边际价值的规模增长。
4. **使用训练集选择阈值**：容易高估模型与策略效果。
5. **反复使用同一份 OOT 调参**：会失去真正的独立验证。
6. **对各箱坏账率直接平均**：忽略样本量和金额暴露差异。
7. **预计收入和违约损失重复扣减**：会造成收益低估。
8. **机械使用 PSI、IV 或固定阈值**：经验规则不能替代统计、业务和经济判断。
9. **仅在历史已通过客户上评估新阈值**：会受到审批选择偏差影响。
10. **模型版本、PD 校准版本和策略版本使用同一个版本号**：难以判断变更来源，也不利于回滚。
11. **用箱内观测最大值或最小值代替真实切点**：线上可能出现边界空档。
12. **将缺失标签填成 0**：会把未成熟或未知样本错误当成好样本。

## 八、执行完成后的最终交付物

每次模型分箱应固定交付：

1. **数据口径表**：来源表、字段、标签、时间和样本量；
2. **最终风险等级表**：真实切点、区间规则、样本量、坏账率、Lift、PD 和 OOT 表现；
3. **累计阈值表**：通过率、累计及边际风险、EL、收入、UE 和审核量；
4. **三套策略方案**：增长、平衡、保守及其约束和压力结果；
5. **上线配置**：模型版本、校准版本、策略版本、生效时间和回滚版本；
6. **验证报告**：调优集、OOT、分群稳定性和已知限制。

最终上线原则：

> **分箱用于建立稳定、可解释的风险等级；阈值用于在风险、规模、收益和审核产能之间做决策。先保证数据口径和边界正确，再优化统计合并，最后接入经济价值进行阈值选择。**
