## 一、核心思路

> **明确风险标签 → 等频初分 → 相邻箱合并 → 跨期验证 → 逐分数绘制阈值曲线 → 在风险、规模和收益约束下确定通过、审核、拒绝阈值。**

需要区分两个概念：

* **分箱**：建立稳定、单调、可解释的风险等级；
* **阈值**：决定哪些客户自动通过、人工审核或拒绝。

阈值可以落在分箱边界上，也可以先在细粒度分数上搜索，再对齐到可上线的分数边界。

---

## 二、风险标签定义

标的逾期率必须同时明确以下三项：

1. **观察期（MOB）**：贷款发放后观察到第几个月；
2. **风险标的**：以客户、申请、借据或放款合同为统计对象；
3. **逾期口径**：逾期天数、时点逾期或观察期内曾经逾期。

---

## 三、分箱方法

### 1. 样本划分

按时间将样本划分为：

* **样本内**：用于设计分箱和选择候选阈值；
* **OOT 样本**：用于最终验证，不能重新分箱或反复调参。


### 2. 等频初分

在样本内按模型分进行**等频 20 箱**：

* 每箱样本量尽量接近；
* 同分较多时允许实际箱数少于 20；
* 首尾边界设置为 `-∞` 和 `+∞`，避免线上样本落在区间之外；

### 3. 计算分箱指标

以下 Python 代码假设已完成等频分箱，DataFrame `df` 包含模型分、风险标签和申请信息字段，并按 `bin` 分组后计算各项指标。所有比率统一输出 `_num`（分子）、`_den`（分母）和最终比率字段。

**统一计算原则：**

- 所有比率使用 `np.where(den > 0, num / den, np.nan)` 避免除零；
- 3M30+、1M30+ 笔数逾期率的分母必须使用对应观察期的**成熟样本量**，不能直接使用箱内总样本量 `n`；
- 金额逾期率的分母使用对应观察期的**成熟样本本金**，避免未达到 MOB1 或 MOB3 的样本稀释逾期率；
- 累计逾期率应累计对应的成熟样本量或成熟本金，不能统一使用 `cum_n` 或 `cum_principal`；
- 从低风险端向高风险端累计时，使用数值型字段 `bin_order` 排序，避免直接按字符型 `bin` 排序发生错位。

```python
import numpy as np
import pandas as pd
from scipy.stats import norm

# ============================================================
# 一、基础规模指标（groupby agg 阶段）
# ============================================================

# n: 箱内总样本量
df['n'] = df.groupby('bin')['application_id'].transform('count')

# total_n: 全部分箱总样本量
df['total_n'] = df['n'].sum()

# sample_pct_num: 箱内样本占比分子
df['sample_pct_num'] = df['n']

# sample_pct_den: 箱内样本占比分母
df['sample_pct_den'] = df['total_n']

# sample_pct: 箱内样本占比
df['sample_pct'] = np.where(
    df['sample_pct_den'] > 0,
    df['sample_pct_num'] / df['sample_pct_den'],
    np.nan
)

# principal_amt: 箱内总放款本金
df['principal_amt'] = df.groupby('bin')['principal'].transform(
    lambda x: x.fillna(0).sum()
)

# ============================================================
# 二、3M30+ 笔数逾期指标
# ============================================================

# 3m30p_cnt_mature: 3M30+ 成熟样本量（duedate_3m_30 可判断为正常/逾期）
df['3m30p_cnt_mature'] = df.groupby('bin')['duedate_3m_30'].transform(
    lambda x: x.isin([0, 1]).sum()
)

# 3m30p_cnt_bad: 3M30+ 标的逾期量（duedate_3m_30 == 1）
df['3m30p_cnt_bad'] = df.groupby('bin')['duedate_3m_30'].transform(
    lambda x: (x == 1).sum()
)

# 3m30p_cnt_good: 3M30+ 成熟正常样本量（duedate_3m_30 == 0，或 mature - bad）
df['3m30p_cnt_good'] = df['3m30p_cnt_mature'] - df['3m30p_cnt_bad']

# 3m30p_cnt_bad_rate_num: 3M30+ 笔数逾期率分子
df['3m30p_cnt_bad_rate_num'] = df['3m30p_cnt_bad']

# 3m30p_cnt_bad_rate_den: 3M30+ 笔数逾期率分母
df['3m30p_cnt_bad_rate_den'] = df['3m30p_cnt_mature']

# 3m30p_cnt_bad_rate: 3M30+ 标的逾期率（笔数口径）
df['3m30p_cnt_bad_rate'] = np.where(
    df['3m30p_cnt_bad_rate_den'] > 0,
    df['3m30p_cnt_bad_rate_num'] / df['3m30p_cnt_bad_rate_den'],
    np.nan
)

# ============================================================
# 三、1M30+ 笔数逾期指标
# ============================================================

df['1m30p_cnt_mature'] = df.groupby('bin')['duedate_1m_30'].transform(
    lambda x: x.isin([0, 1]).sum()
)

df['1m30p_cnt_bad'] = df.groupby('bin')['duedate_1m_30'].transform(
    lambda x: (x == 1).sum()
)

df['1m30p_cnt_good'] = df['1m30p_cnt_mature'] - df['1m30p_cnt_bad']

df['1m30p_cnt_bad_rate_num'] = df['1m30p_cnt_bad']
df['1m30p_cnt_bad_rate_den'] = df['1m30p_cnt_mature']

df['1m30p_cnt_bad_rate'] = np.where(
    df['1m30p_cnt_bad_rate_den'] > 0,
    df['1m30p_cnt_bad_rate_num'] / df['1m30p_cnt_bad_rate_den'],
    np.nan
)

# ============================================================
# 四、3M30+ 金额逾期指标
# ============================================================

# 3m30p_amt_exposure: 3M30+ 成熟样本对应的本金（金额逾期率分母）
df['3m30p_amt_exposure'] = df.groupby('bin').apply(
    lambda g: g.loc[g['duedate_3m_30'].isin([0, 1]), 'principal'].fillna(0).sum()
).reindex(df['bin']).values

# 3m30p_amt_bad: 3M30+ 金额逾期量（MOB3 时历史最大 dpd >= 30 的剩余本金）
df['3m30p_amt_bad'] = df.groupby('bin').apply(
    lambda g: g.loc[
        g['duedate_3m_30'].isin([0, 1]) & (g['dpd_days_ever_mob3'] >= 30),
        'estimate_principal_remaining_mob3'
    ].fillna(0).sum()
).reindex(df['bin']).values

# 3m30p_amt_bad_cnt: 发生 3M30+ 金额逾期的样本数
df['3m30p_amt_bad_cnt'] = df.groupby('bin').apply(
    lambda g: (
        g['duedate_3m_30'].isin([0, 1]) & (g['dpd_days_ever_mob3'] >= 30)
    ).sum()
).reindex(df['bin']).values

df['3m30p_amt_bad_rate_num'] = df['3m30p_amt_bad']
df['3m30p_amt_bad_rate_den'] = df['3m30p_amt_exposure']

df['3m30p_amt_bad_rate'] = np.where(
    df['3m30p_amt_bad_rate_den'] > 0,
    df['3m30p_amt_bad_rate_num'] / df['3m30p_amt_bad_rate_den'],
    np.nan
)

# ============================================================
# 五、1M30+ 金额逾期指标
# ============================================================

df['1m30p_amt_exposure'] = df.groupby('bin').apply(
    lambda g: g.loc[g['duedate_1m_30'].isin([0, 1]), 'principal'].fillna(0).sum()
).reindex(df['bin']).values

df['1m30p_amt_bad'] = df.groupby('bin').apply(
    lambda g: g.loc[
        g['duedate_1m_30'].isin([0, 1]) & (g['dpd_days_ever_mob1'] >= 30),
        'estimate_principal_remaining_mob1'
    ].fillna(0).sum()
).reindex(df['bin']).values

df['1m30p_amt_bad_cnt'] = df.groupby('bin').apply(
    lambda g: (
        g['duedate_1m_30'].isin([0, 1]) & (g['dpd_days_ever_mob1'] >= 30)
    ).sum()
).reindex(df['bin']).values

df['1m30p_amt_bad_rate_num'] = df['1m30p_amt_bad']
df['1m30p_amt_bad_rate_den'] = df['1m30p_amt_exposure']

df['1m30p_amt_bad_rate'] = np.where(
    df['1m30p_amt_bad_rate_den'] > 0,
    df['1m30p_amt_bad_rate_num'] / df['1m30p_amt_bad_rate_den'],
    np.nan
)

# ============================================================
# 六、整体逾期率与 Lift（基于汇总后的 bin_stats 表计算）
# ============================================================

# 以下假设已经从 df 中提取每个 bin 聚合后的 bin_stats DataFrame
# bin_stats 包含每箱一行，含以上所有 _cnt_ 和 _amt_ 指标

# --- 整体 3M30+ 笔数逾期率 ---
overall_3m30p_cnt_bad_rate_num = bin_stats['3m30p_cnt_bad'].sum()
overall_3m30p_cnt_bad_rate_den = bin_stats['3m30p_cnt_mature'].sum()
overall_3m30p_cnt_bad_rate = (
    overall_3m30p_cnt_bad_rate_num
    / overall_3m30p_cnt_bad_rate_den
    if overall_3m30p_cnt_bad_rate_den > 0 else np.nan
)

# 3m30p_cnt_lift: 各箱笔数逾期率 / 整体笔数逾期率
bin_stats['3m30p_cnt_lift_num'] = bin_stats['3m30p_cnt_bad_rate']
bin_stats['3m30p_cnt_lift_den'] = overall_3m30p_cnt_bad_rate
bin_stats['3m30p_cnt_lift'] = np.where(
    bin_stats['3m30p_cnt_lift_den'] > 0,
    bin_stats['3m30p_cnt_lift_num'] / bin_stats['3m30p_cnt_lift_den'],
    np.nan
)

# --- 整体 1M30+ 笔数逾期率 ---
overall_1m30p_cnt_bad_rate_num = bin_stats['1m30p_cnt_bad'].sum()
overall_1m30p_cnt_bad_rate_den = bin_stats['1m30p_cnt_mature'].sum()
overall_1m30p_cnt_bad_rate = (
    overall_1m30p_cnt_bad_rate_num
    / overall_1m30p_cnt_bad_rate_den
    if overall_1m30p_cnt_bad_rate_den > 0 else np.nan
)

bin_stats['1m30p_cnt_lift_num'] = bin_stats['1m30p_cnt_bad_rate']
bin_stats['1m30p_cnt_lift_den'] = overall_1m30p_cnt_bad_rate
bin_stats['1m30p_cnt_lift'] = np.where(
    bin_stats['1m30p_cnt_lift_den'] > 0,
    bin_stats['1m30p_cnt_lift_num'] / bin_stats['1m30p_cnt_lift_den'],
    np.nan
)

# --- 整体 3M30+ 金额逾期率 ---
overall_3m30p_amt_bad_rate_num = bin_stats['3m30p_amt_bad'].sum()
overall_3m30p_amt_bad_rate_den = bin_stats['3m30p_amt_exposure'].sum()
overall_3m30p_amt_bad_rate = (
    overall_3m30p_amt_bad_rate_num
    / overall_3m30p_amt_bad_rate_den
    if overall_3m30p_amt_bad_rate_den > 0 else np.nan
)

bin_stats['3m30p_amt_lift_num'] = bin_stats['3m30p_amt_bad_rate']
bin_stats['3m30p_amt_lift_den'] = overall_3m30p_amt_bad_rate
bin_stats['3m30p_amt_lift'] = np.where(
    bin_stats['3m30p_amt_lift_den'] > 0,
    bin_stats['3m30p_amt_lift_num'] / bin_stats['3m30p_amt_lift_den'],
    np.nan
)

# --- 整体 1M30+ 金额逾期率 ---
overall_1m30p_amt_bad_rate_num = bin_stats['1m30p_amt_bad'].sum()
overall_1m30p_amt_bad_rate_den = bin_stats['1m30p_amt_exposure'].sum()
overall_1m30p_amt_bad_rate = (
    overall_1m30p_amt_bad_rate_num
    / overall_1m30p_amt_bad_rate_den
    if overall_1m30p_amt_bad_rate_den > 0 else np.nan
)

bin_stats['1m30p_amt_lift_num'] = bin_stats['1m30p_amt_bad_rate']
bin_stats['1m30p_amt_lift_den'] = overall_1m30p_amt_bad_rate
bin_stats['1m30p_amt_lift'] = np.where(
    bin_stats['1m30p_amt_lift_den'] > 0,
    bin_stats['1m30p_amt_lift_num'] / bin_stats['1m30p_amt_lift_den'],
    np.nan
)

# ============================================================
# 七、笔数逾期率标准误和置信区间
# ============================================================

# 笔数逾期率属于二项比例，标准误的分母使用对应的成熟样本量（不是箱内总样本量 n）。
# 当成熟样本量较小或逾期率接近 0、1 时，建议使用 Wilson 置信区间代替普通正态近似。

# --- 3M30+ 正态近似置信区间 ---
bin_stats['3m30p_cnt_bad_rate_se'] = np.sqrt(
    bin_stats['3m30p_cnt_bad_rate']
    * (1 - bin_stats['3m30p_cnt_bad_rate'])
    / np.where(bin_stats['3m30p_cnt_mature'] > 0, bin_stats['3m30p_cnt_mature'], np.nan)
)

z = norm.ppf(0.975)  # 1.96
bin_stats['3m30p_cnt_bad_rate_ci_lower'] = np.clip(
    bin_stats['3m30p_cnt_bad_rate'] - z * bin_stats['3m30p_cnt_bad_rate_se'],
    0, None
)
bin_stats['3m30p_cnt_bad_rate_ci_upper'] = np.clip(
    bin_stats['3m30p_cnt_bad_rate'] + z * bin_stats['3m30p_cnt_bad_rate_se'],
    None, 1
)

# --- 1M30+ 正态近似置信区间 ---
bin_stats['1m30p_cnt_bad_rate_se'] = np.sqrt(
    bin_stats['1m30p_cnt_bad_rate']
    * (1 - bin_stats['1m30p_cnt_bad_rate'])
    / np.where(bin_stats['1m30p_cnt_mature'] > 0, bin_stats['1m30p_cnt_mature'], np.nan)
)

bin_stats['1m30p_cnt_bad_rate_ci_lower'] = np.clip(
    bin_stats['1m30p_cnt_bad_rate'] - z * bin_stats['1m30p_cnt_bad_rate_se'],
    0, None
)
bin_stats['1m30p_cnt_bad_rate_ci_upper'] = np.clip(
    bin_stats['1m30p_cnt_bad_rate'] + z * bin_stats['1m30p_cnt_bad_rate_se'],
    None, 1
)

# --- Wilson 置信区间（推荐用于小样本或极端比率） ---
def wilson_ci(numerator, denominator, alpha=0.05):
    """Wilson score interval for a binomial proportion."""
    z = norm.ppf(1 - alpha / 2)
    p = numerator / denominator
    denom = 1 + z**2 / denominator
    center = (p + z**2 / (2 * denominator)) / denom
    margin = z * np.sqrt((p * (1 - p) + z**2 / (4 * denominator)) / denominator) / denom
    return center - margin, center + margin
```

### 4. 合并相邻箱

按以下顺序合并：

1. 样本量或标的逾期量过少的箱；
2. 主标签标的逾期率明显倒挂且差异不显著的箱；
3. 相邻箱风险分布差异不显著的箱；
4. 3M30+ 与 1M30+ 风险排序明显冲突的箱；
5. 跨月份或 OOT 表现不稳定的箱。

可使用 **ChiMerge、两比例检验和标的逾期率置信区间**辅助判断。

一般以 **3M30+** 的单调性作为主要合并依据，同时检查 **1M30+** 是否出现明显异常。若两个标签方向长期冲突，应进一步检查：

* 样本成熟度；
* 标签定义；
* 早期欺诈或首逾客群；
* 产品、渠道及客群结构差异；
* 数据质量和还款计划异常。

最终一般保留 **6～10 个风险等级**，并满足：

* 3M30+ 标的逾期率整体单调；
* 1M30+ 风险趋势基本合理；
* 单箱样本量和标的逾期量充分；
* 相邻等级存在可识别的风险差异；
* OOT 风险排序基本稳定；
* 边界便于解释和系统配置。

## 四、阈值曲线方法

假设模型为**高分高风险**，候选阈值为 (c)，则通过人群为：

[
score \leq c
]

候选阈值可以使用：

* 每个模型分唯一值；
* 固定分数步长；
* 分数分位点；
* 最终分箱边界。

对每个候选阈值，从低风险端向高风险端累计计算：

| 指标                  | 作用         |
| ------------------- | ---------- |
| 累计通过率               | 衡量业务规模     |
| 累计 3M30+ 标的逾期率      | 笔数口径中短期信用风险 |
| 累计 1M30+ 标的逾期率      | 笔数口径早期逾期风险 |
| 边际 3M30+ 标的逾期率      | 新增客群笔数口径中短期风险 |
| 边际 1M30+ 标的逾期率      | 新增客群笔数口径早期风险 |
| 累计 3M30+ 金额逾期率      | 本金口径中短期信用风险 |
| 累计 1M30+ 金额逾期率      | 本金口径早期逾期风险 |
| EL                  | 衡量预期损失     |
| 风险后收入或 UE           | 衡量经济价值     |
| 审核量                 | 衡量人工产能压力   |

其中，**边际风险**非常重要。当阈值继续放宽时，如果新增客群的 3M30+ 或 1M30+ 标的逾期率快速上升，且边际收益转负，通常意味着已经接近合理阈值。

两类风险指标的使用方式应有所区别：

* **3M30+**：主要用于控制整体信用风险、损失和 EL；
* **1M30+**：主要用于识别首逾、欺诈、早期失联或还款能力异常；
* 当 1M30+ 较高但 3M30+ 尚可时，可优先考虑人工审核、降低额度或增加验证，而不一定直接拒绝。

## 五、策略阈值确定

采用三段式策略：

```text
自动通过：score <= 自动通过阈值
人工审核：自动通过阈值 < score <= 拒绝阈值
拒绝：score > 拒绝阈值
```

阈值不能仅按箱数比例机械切分，而应在以下约束下优化：

* 3M30+ 标的逾期率不超过风险上限；
* 1M30+ 标的逾期率不超过早期风险上限；
* EL 或金额逾期率处于可接受范围；
* 通过率或放款规模达到业务目标；
* 风险后收入或总 UE 不低于经营底线；
* 新增客群的边际 UE 为正；
* 人工审核量不超过团队产能；
* OOT 和压力情景下仍可接受。

最终建议输出三套方案：

| 方案   | 主要目标                         |
| ---- | ---------------------------- |
| 增长方案 | 在 3M30+ 和 1M30+ 风险底线内最大化通过率 |
| 平衡方案 | 平衡通过率、中短期风险、早期风险和收益          |
| 保守方案 | 优先控制 3M30+、1M30+ 和预期损失      |

正式推荐方案应选择满足约束且位于**风险—规模—收益 Pareto 前沿**的阈值组合，而不是只追求最低标的逾期率或最高通过率。

## 六、验证与上线

最终分箱和阈值必须在 OOT 上复用完全相同的边界，重点验证：

* 各风险等级的 3M30+ 排序是否稳定；
* 各风险等级的 1M30+ 排序是否合理；
* 通过率、两类标的逾期率和 EL 是否符合预期；
* PSI、AUC、KS 是否出现明显恶化；
* 不同月份、产品、渠道和客群是否存在局部失效；
* 人工审核量是否可执行；
* 3M30+ 与 1M30+ 的风险关系是否发生明显变化。

最终上线配置应明确：

* 模型版本；
* 分数方向；
* 分箱真实切点；
* 区间左右开闭规则；
* 自动通过和拒绝阈值；
* 主风险标签和辅助风险标签；
* 风险标的是客户、申请还是借据；
* 3M30+ 的观察期及逾期定义；
* 1M30+ 的观察期及逾期定义；
* 标签成熟条件及样本时间范围；
* 生效时间和回滚版本。

建议将标签口径独立配置，避免后续出现同名指标定义不一致的问题：

| 配置项      | 示例              |
| -------- | --------------- |
| 风险标的     | 借据              |
| 主标签      | 3M30+        |
| 主标签观察期   | 放款后 0～3M   |
| 主标签逾期口径  | 观察期内曾达到 DPD 30+ |
| 辅助标签     | 1M30+             |
| 辅助标签观察期  | 首期或放款后第一个月      |
| 辅助标签逾期口径 | 观察期内曾达到 DPD 5+  |

以上仅为配置示例，最终应以公司正式业务定义为准。

## 七、一句话总结
