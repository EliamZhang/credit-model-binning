## 一、核心思路

> **明确风险标签 → 等频初分 → 相邻箱合并 → 跨期验证 → 逐分数绘制阈值曲线 → 在风险、规模和收益约束下确定通过、审核、拒绝阈值。**

需要区分两个概念：

* **分箱**：建立稳定、单调、可解释的风险等级；
* **阈值**：决定哪些客户自动通过、人工审核或拒绝。

阈值可以落在分箱边界上，也可以先在细粒度分数上搜索，再对齐到可上线的分数边界。

---

## 二、风险标签定义

标的逾期率必须同时明确以下三项：

1. **观察期（MOB）**：贷款发放后观察到第几个月；
2. **风险标的**：以客户、申请、借据或放款合同为统计对象；
3. **逾期口径**：逾期天数、时点逾期或观察期内曾经逾期。

---

## 三、分箱方法

### 1. 样本划分

按时间将样本划分为：

* **样本内**：用于设计分箱和选择候选阈值；
* **OOT 样本**：用于最终验证，不能重新分箱或反复调参。


### 2. 等频初分

在样本内按模型分进行**等频 20 箱**：

* 每箱样本量尽量接近；
* 同分较多时允许实际箱数少于 20；
* 首尾边界设置为 `-∞` 和 `+∞`，避免线上样本落在区间之外；

### 3. 计算分箱指标

每箱计算：

* `n`：样本量
* `sample_pct`：样本占比
* `3m30p_cnt_bad`：3M30+ 标的逾期量
* `3m30p_cnt_bad_rate`：3M30+ 标的逾期率（笔数口径）
* `1m30p_cnt_bad`：1M30+ 标的逾期量
* `1m30p_cnt_bad_rate`：1M30+ 标的逾期率（笔数口径）
* `3m30p_amt_bad`：3M30+ 金额逾期量
* `3m30p_amt_bad_rate`：3M30+ 金额逾期率（本金口径）
* `1m30p_amt_bad`：1M30+ 金额逾期量
* `1m30p_amt_bad_rate`：1M30+ 金额逾期率（本金口径）
* `3m30p_cnt_lift`：3M30+ 标的 Lift
* `1m30p_cnt_lift`：1M30+ 标的 Lift
* `3m30p_amt_lift`：3M30+ 金额 Lift
* `1m30p_amt_lift`：1M30+ 金额 Lift
* `cnt_bad_rate_se`：标的逾期率标准误
* `cnt_bad_rate_ci`：标的逾期率置信区间
* `amt_bad_rate_se`：金额逾期率标准误
* `amt_bad_rate_ci`：金额逾期率置信区间
* `cum_pass_rate`：累计通过率
* `cum_3m30p_cnt_bad_rate`：累计 3M30+ 标的逾期率
* `cum_3m30p_amt_bad_rate`：累计 3M30+ 金额逾期率
* `cum_1m30p_cnt_bad_rate`：累计 1M30+ 标的逾期率
* `cum_1m30p_amt_bad_rate`：累计 1M30+ 金额逾期率


### 4. 合并相邻箱

按以下顺序合并：

1. 样本量或标的逾期量过少的箱；
2. 主标签标的逾期率明显倒挂且差异不显著的箱；
3. 相邻箱风险分布差异不显著的箱；
4. 3M30+ 与 1M30+ 风险排序明显冲突的箱；
5. 跨月份或 OOT 表现不稳定的箱。

可使用 **ChiMerge、两比例检验和标的逾期率置信区间**辅助判断。

一般以 **3M30+** 的单调性作为主要合并依据，同时检查 **1M30+** 是否出现明显异常。若两个标签方向长期冲突，应进一步检查：

* 样本成熟度；
* 标签定义；
* 早期欺诈或首逾客群；
* 产品、渠道及客群结构差异；
* 数据质量和还款计划异常。

最终一般保留 **6～10 个风险等级**，并满足：

* 3M30+ 标的逾期率整体单调；
* 1M30+ 风险趋势基本合理；
* 单箱样本量和标的逾期量充分；
* 相邻等级存在可识别的风险差异；
* OOT 风险排序基本稳定；
* 边界便于解释和系统配置。

## 四、阈值曲线方法

假设模型为**高分高风险**，候选阈值为 (c)，则通过人群为：

[
score \leq c
]

候选阈值可以使用：

* 每个模型分唯一值；
* 固定分数步长；
* 分数分位点；
* 最终分箱边界。

对每个候选阈值，从低风险端向高风险端累计计算：

| 指标                  | 作用         |
| ------------------- | ---------- |
| 累计通过率               | 衡量业务规模     |
| 累计 3M30+ 标的逾期率      | 笔数口径中短期信用风险 |
| 累计 1M30+ 标的逾期率      | 笔数口径早期逾期风险 |
| 边际 3M30+ 标的逾期率      | 新增客群笔数口径中短期风险 |
| 边际 1M30+ 标的逾期率      | 新增客群笔数口径早期风险 |
| 累计 3M30+ 金额逾期率      | 本金口径中短期信用风险 |
| 累计 1M30+ 金额逾期率      | 本金口径早期逾期风险 |
| EL                  | 衡量预期损失     |
| 风险后收入或 UE           | 衡量经济价值     |
| 审核量                 | 衡量人工产能压力   |

其中，**边际风险**非常重要。当阈值继续放宽时，如果新增客群的 3M30+ 或 1M30+ 标的逾期率快速上升，且边际收益转负，通常意味着已经接近合理阈值。

两类风险指标的使用方式应有所区别：

* **3M30+**：主要用于控制整体信用风险、损失和 EL；
* **1M30+**：主要用于识别首逾、欺诈、早期失联或还款能力异常；
* 当 1M30+ 较高但 3M30+ 尚可时，可优先考虑人工审核、降低额度或增加验证，而不一定直接拒绝。

## 五、策略阈值确定

采用三段式策略：

```text
自动通过：score <= 自动通过阈值
人工审核：自动通过阈值 < score <= 拒绝阈值
拒绝：score > 拒绝阈值
```

阈值不能仅按箱数比例机械切分，而应在以下约束下优化：

* 3M30+ 标的逾期率不超过风险上限；
* 1M30+ 标的逾期率不超过早期风险上限；
* EL 或金额逾期率处于可接受范围；
* 通过率或放款规模达到业务目标；
* 风险后收入或总 UE 不低于经营底线；
* 新增客群的边际 UE 为正；
* 人工审核量不超过团队产能；
* OOT 和压力情景下仍可接受。

最终建议输出三套方案：

| 方案   | 主要目标                         |
| ---- | ---------------------------- |
| 增长方案 | 在 3M30+ 和 1M30+ 风险底线内最大化通过率 |
| 平衡方案 | 平衡通过率、中短期风险、早期风险和收益          |
| 保守方案 | 优先控制 3M30+、1M30+ 和预期损失      |

正式推荐方案应选择满足约束且位于**风险—规模—收益 Pareto 前沿**的阈值组合，而不是只追求最低标的逾期率或最高通过率。

## 六、验证与上线

最终分箱和阈值必须在 OOT 上复用完全相同的边界，重点验证：

* 各风险等级的 3M30+ 排序是否稳定；
* 各风险等级的 1M30+ 排序是否合理；
* 通过率、两类标的逾期率和 EL 是否符合预期；
* PSI、AUC、KS 是否出现明显恶化；
* 不同月份、产品、渠道和客群是否存在局部失效；
* 人工审核量是否可执行；
* 3M30+ 与 1M30+ 的风险关系是否发生明显变化。

最终上线配置应明确：

* 模型版本；
* 分数方向；
* 分箱真实切点；
* 区间左右开闭规则；
* 自动通过和拒绝阈值；
* 主风险标签和辅助风险标签；
* 风险标的是客户、申请还是借据；
* 3M30+ 的观察期及逾期定义；
* 1M30+ 的观察期及逾期定义；
* 标签成熟条件及样本时间范围；
* 生效时间和回滚版本。

建议将标签口径独立配置，避免后续出现同名指标定义不一致的问题：

| 配置项      | 示例              |
| -------- | --------------- |
| 风险标的     | 借据              |
| 主标签      | 3M30+        |
| 主标签观察期   | 放款后 0～3M   |
| 主标签逾期口径  | 观察期内曾达到 DPD 30+ |
| 辅助标签     | 1M30+             |
| 辅助标签观察期  | 首期或放款后第一个月      |
| 辅助标签逾期口径 | 观察期内曾达到 DPD 5+  |

以上仅为配置示例，最终应以公司正式业务定义为准。

---

## 七、一句话总结

> **分箱负责将模型分整理成稳定的风险等级；阈值负责沿分数逐步放宽准入，并结合 3M30+、1M30+、规模、收益和审核产能寻找最优决策点。**
